一条隧道，把你的电脑变成免费网站

从 TXT 到可分享网页的全过程

一个真实案例

有人需要把一个 TXT 文档做成网页，发给朋友直接能看。于是就这么干了：

第一步，读内容。 找到电脑上的 TXT 文件，把内容读出来。

第二步，生成网页。 用 Python 脚本把纯文本转成一个暗色主题的 HTML 页面——有标题、有代码块、有表格，手机电脑打开都好看。

第三步，本地起服务。 python3 -m http.server 3458，本机 3458 端口开始监听。

第四步，拉隧道。 运行 cloudflared tunnel --url http://localhost:3458，电脑主动向 Cloudflare 服务器发出一条加密连接，像一根隐形的管子，一头连着本机，一头连着 Cloudflare 的全球网络。Cloudflare 分配一个随机域名。

第五步，分享。 把链接发给朋友，朋友浏览器一打开，请求先到 Cloudflare，Cloudflare 穿过隧道找到那台电脑上的 HTML 文件，原路返回。朋友看到的就是一个完整的网页。

全程没有注册任何账号、没有买服务器、没有配置域名、没有开端口、没有公网 IP。就三个东西在跑：一个 HTML 文件、一个 Python 进程、一个 cloudflared 进程。

这根隧道的技术细节

它用的是 QUIC 协议——就是 HTTP/3 的底层那个，基于 UDP。传统 TCP 连接要三次握手才能开始传数据，QUIC 可以 0-RTT，发第一个包就直接带数据，延迟更低。而且 QUIC 自带加密，连接层就有一层加密，传的 HTTP 流量本身又有一层 HTTPS，双重加密。UDP 不通的话会自动降级 TCP，支持 IPv4/IPv6 双栈。

说白了就是：电脑主动向外连 Cloudflare，Cloudflare 当中转站，全世界任何人点一下链接就能看到这台电脑上的内容。 关掉进程，链接就消失，干干净净。

不只是这一篇文档

这个场景只是冰山一角——飞书文档、Markdown 笔记、代码片段、任何文字内容，都可以用同样的方式一键变成可分享的网页。

有人把这个流程固化成了一个 OpenClaw Skill，以后说"做成网页"，就自动走完这套流程，连按钮都不用点。
