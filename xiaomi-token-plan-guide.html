小米 Token Plan 接入龙虾（OpenClaw）简要总结

一、最容易踩的坑

  1. 不能混用按量付费和 Token Plan
  • 按量付费：sk-... + https://api.xiaomimimo.com/v1
  • Token Plan：tp-... + https://token-plan-cn.xiaomimimo.com/v1
  1. Token Plan 不能走 OpenClaw 交互式向导
  • 官方要求直接修改 ~/.openclaw/openclaw.json
  1. auth 段要删掉
  • Token Plan 配置时不要保留 "auth"
  1. 模型 ID 必须写小写
  • 正确：mimo-v2-pro
  • 不要写：MiMo-V2-Pro

二、最小正确配置

{

"models": {

"mode": "merge",

"providers": {

"xiaomi": {

"baseUrl": "https://token-plan-cn.xiaomimimo.com/v1",

"apiKey": "tp-你的TokenPlanKey",

"api": "openai-completions",

"models": [

{

"id": "mimo-v2-pro",

"name": "mimo-v2-pro",

"reasoning": true,

"input": ["text"],

"contextWindow": 1048576,

"maxTokens": 32000

}

]

}

}

},

"agents": {

"defaults": {

"model": {

"primary": "xiaomi/mimo-v2-pro",

"fallbacks": []

},

"models": {

"xiaomi/mimo-v2-pro": {

"alias": "MiMo V2 Pro"

}

}

}

}

}

三、一句话记忆

Token Plan = tp-key + token-plan 专属 baseUrl + 手改 openclaw.json + 删除 auth + 模型名用 mimo-v2-pro
